1. O app deve ser construido com a metodologia Feature-First (Por Funcionalidade). Aqui, cada "funcionalidade" do app ganha sua própria pasta com tudo o que ela precisa dentro.

2. O app deve ser bastante modular e organizado. Sempre que for adicionar uma nova funcionalidade, deve-se criar uma nova pasta e adicionar tudo o que ela precisa dentro.

3. O app deve ser organizado em camadas, cada uma com sua responsabilidade. 

4. O app deve ser simples, limpo e organizado. Sempre prezando pelo minimalismo e pela usabilidade.

5. Para cada escrita de código, deve ser feita em português. 

6. Para cada escrita de código, deve-se escrever o básico possível e se possível quebrar as funcionalidades em funções menores e arquivos menores.

7. Para cada escrita de código, deve-se adicionar um comentario explicando o que foi feito e por que foi feito. (O comentário deve estar em português).

8. Para cada arquivo criado, deve-se adicionar um comentario no topo do arquivo explicando o que é o arquivo. (O comentário deve estar em português).

9. O nome dos arquivos devem ser em portugues. Usando a nomenclatura snake_case.
